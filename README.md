````markdown
# Gulp Frontend Build System

Автоматизированная сборка для фронтенд-разработки с поддержкой SCSS, JavaScript (ES6+), SVG-спрайтов, оптимизации изображений и BrowserSync.

## 📋 Требования

- [Node.js](https://nodejs.org/) (версия 14.x или выше)
- [npm](https://www.npmjs.com/) (устанавливается вместе с Node.js)
- [Gulp CLI](https://gulpjs.com/) (глобально)

## 🚀 Быстрый старт

### 1. Установка Gulp CLI (глобально)

```bash
npm install --global gulp-cli
```
````

### 2. Установка зависимостей проекта

Перейдите в папку с проектом и выполните:

```bash
npm install
```

### 3. Настройка BrowserSync (для WordPress)

Откройте `gulpfile.js` и найдите строку:

```javascript
proxy: "http://primasnab.loc", // ИЗМЕНИТЕ НА ВАШ ЛОКАЛЬНЫЙ URL
```

Замените на URL вашего локального WordPress сайта.

### 4. Запуск сборки

```bash
gulp          # Режим разработки (по умолчанию)
gulp dev      # То же самое
gulp build    # Сборка для продакшена
gulp backend  # Сборка для бэкенда (без sourcemaps)
```

## 🔧 Режимы сборки

### Режим разработки (`gulp dev`)

- ✅ Sourcemaps для CSS и JS
- ✅ Не минифицированный код (легко отлаживать)
- ✅ Автоматическое обновление браузера (BrowserSync)
- ✅ Отслеживание изменений в файлах
- ✅ Оптимизация изображений в реальном времени

**Используйте для:** локальной разработки

### Режим продакшена (`gulp build`)

- ❌ Sourcemaps отключены
- ✅ Минификация CSS (CleanCSS)
- ✅ Минификация JS (UglifyJS)
- ✅ Сжатие изображений (Imagemin)
- ✅ Конвертация в WebP

**Используйте для:** финальной сборки перед выкладкой на сервер

### Режим бэкенда (`gulp backend`)

- ❌ Sourcemaps отключены
- ❌ Без минификации (для совместимости)
- ✅ Оптимизация изображений

**Используйте для:** выкладки на staging-сервер или при проблемах с минификацией

## 📝 Команды

| Команда               | Описание                                     |
| --------------------- | -------------------------------------------- |
| `gulp` или `gulp dev` | Запуск разработки с автообновлением          |
| `gulp build`          | Сборка для продакшена                        |
| `gulp backend`        | Сборка для бэкенда                           |
| `gulp watch`          | Только отслеживание файлов (без BrowserSync) |
| `gulp styles`         | Собрать только CSS                           |
| `gulp scripts`        | Собрать только JS                            |
| `gulp svgsprite`      | Собрать только SVG-спрайт                    |
| `gulp images`         | Оптимизировать изображения                   |
| `gulp webp`           | Конвертировать изображения в WebP            |

## 🎯 Что делает сборка

### SCSS → CSS

- Компиляция SCSS в CSS с помощью `sass` (Dart Sass)
- Автопрефиксы для поддержки последних 5 версий браузеров
- Grid support включен
- Sourcemaps в режиме разработки
- Минификация в продакшене

### JavaScript (ES6+ → ES5)

- Сборка модулей через Webpack
- Транспиляция через Babel (preset-env)
- Поддержка современных браузеров (targets: "defaults")
- Sourcemaps в режиме разработки
- Минификация в продакшене

### SVG-спрайт

- Автоматическая сборка всех SVG из `img/svg/`
- Очистка от атрибутов `fill`, `stroke`, `style`
- Создание единого файла `img/sprite.svg`

**Использование в HTML:**

```html
<svg>
  <use href="img/sprite.svg#icon-name"></use>
</svg>
```

### Оптимизация изображений

- Сжатие JPG (mozjpeg, качество 80%)
- Сжатие PNG (optipng, уровень 2)
- Конвертация в формат WebP
- Обработка только новых файлов (gulp-newer)

## 🔄 BrowserSync

При запуске `gulp dev` автоматически:

1. Запускается локальный сервер с проксированием вашего WordPress сайта
2. Страница автоматически обновляется при изменении:
   - SCSS (инжектит CSS без перезагрузки)
   - JS (полная перезагрузка)
   - SVG (пересборка спрайта)
   - PHP (полная перезагрузка)
   - Изображений (перезагрузка)

**Порт:** автоматический (обычно 3000)  
**URL:** `http://localhost:3000`

## 🛠️ Решение проблем

### Ошибка "gulp: command not found"

```bash
npm install --global gulp-cli
```

### Ошибка при установке зависимостей

```bash
rm -rf node_modules package-lock.json
npm install
```

### BrowserSync не подключается к WordPress

1. Проверьте URL в `gulpfile.js` (строка с `proxy:`)
2. Убедитесь, что локальный WordPress работает
3. Проверьте настройки `hosts` файла

### Sourcemaps не создаются

- Режим разработки: `gulp dev`
- В режимах `build` и `backend` sourcemaps отключены намеренно

### Изображения не оптимизируются

- Добавьте новые изображения в папку `img/`
- Сборка обрабатывает только новые файлы (для производительности)
- Принудительная оптимизация: удалите файлы из `img/` и запустите `gulp images`

### SCSS ошибки

Сборка показывает уведомление с текстом ошибки, но продолжает работу.
Проверьте синтаксис SCSS и импорты.

- `gulp-plumber` - обработка ошибок
- `gulp-notify` - уведомления об ошибках
- `del` - удаление файлов
- `gulp-newer` - обработка только новых файлов
